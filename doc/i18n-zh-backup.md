# Demo 期間「強制日文」中文備份

本文件記錄 demo 階段被 `tUI()` 包覆的所有中文字串（key 即原中文，因此 `frontend/src/lib/uiStrings.ts` 本身已是字典源頭）。
這份文件作為「之後接 react-i18next 時的詞庫起點」與「掃描檢查清單」。

## 恢復中文

最快做法：把 `frontend/src/lib/forcedLanguage.ts` 的 `FORCED_LANGUAGE` 改為 `null`，所有 `tUI()` 立刻 fallback 回中文，無需動其他檔案。

完全移除（demo 結束、改用正式 i18n 框架）：
1. 刪除 `frontend/src/lib/forcedLanguage.ts`
2. 刪除 `frontend/src/lib/uiStrings.ts`
3. 全文搜尋 `import { tUI }` / `import { FORCED_LANGUAGE }` 並還原為 hardcode 中文
4. 還原 `frontend/src/features/check-in/components/Calendar.tsx` 的 WEEKDAYS 為原本陣列

## 涵蓋範圍 / 不涵蓋

涵蓋：Header、登入/註冊、首頁、簽到、現金、儲蓄、股票、保險、排行榜、learn 模組外殼（CourseList/Detail/Chapter）。

不涵蓋（刻意保留中文）：
- `frontend/src/features/admin/*` — 後台不需翻譯
- learn 模組 lesson 內容（章節 pill、unit 名、quiz 題目、story 選項、game HTML、結算卡）— 仍依 `parseLessonKey()` 解析出的 `language` 顯示，由 `gameTranslations.ts` 既有的 `t()` / `getChapterTypeLabel()` / `getUnitNameI18n()` / `translateGameHtml()` 控管
- Firestore fallback 等級名稱（`打工新手` / `資深打工族`，僅在 Firestore 斷線時出現）
- 註解、變數命名、log 訊息

## 例外：中文版第一課

`/course/stage-1-lesson-1`（無 `-ja` suffix）：lesson 內容仍是中文，但**外層 Header 會跟 `FORCED_LANGUAGE` 走（即顯示日文）**。這是方案 A 的設計取捨——避免動 React Context 與 routing。
