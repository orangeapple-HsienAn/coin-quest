# 財商學習平台 規格文件

## 1. 平台概述

本平台是結合「遊戲化」與「模擬交易」的財商學習平台。使用者能在平台上進行課程，並透過學習課程與每日互動獲得虛擬貨幣。再利用該貨幣進行擬真的投資、儲蓄與保險規劃，藉此建立正確的理財觀念。

## 2. 全域數值與經濟系統

### 2.1 核心數值

- 金幣：平台通用貨幣
  - 用途：投資股票、銀行儲蓄、購買保險、支付風險事件費用
  - 來源：每日簽到薪資、任務獎勵、投資獲利、課程完成獎勵
- 經驗值：決定使用者等級
  - 來源：觀看教材、回答測驗
  - 用途：累積經驗值可提升「等級」

### 2.2 等級與薪資制度

## 3. 首頁儀表板

### 3.1 任務區塊

每日固定顯示三項任務：

| 任務 | 獎勵 | 點擊行為 |
|------|------|----------|
| 每日測驗 | +200 金幣 | 開始進行測驗，每日限一次 |
| 變動任務 | +300 金幣 | 隨機顯示購買股票/儲蓄/保險等任務，跳轉對應頁面 |
| 課程任務 | +3,000 金幣 | 進入課程頁面，完成一堂課程即完成 |

狀態顯示：
- 未完成：顯示圓圈，支援點擊進入對應功能
- 已完成：顯示打勾圖示，外框變綠，不可再次點擊

### 3.2 快捷功能區

- **簽到按鈕**：顯示「簽到」、「日薪金額」與右側箭頭，點擊進入簽到頁面
  - 未完成：底色與框為紅色
  - 已完成：底色與框為綠色，仍可點擊進入簽到頁面
- **課程入口卡片**：顯示大圖與標題，點擊跳轉至課程首頁

### 3.3 資產儀表板

- 資產卡片：顯示資產總額
- 排名按鈕：顯示目前排名，支援點擊跳轉至排行榜頁面
- 圓餅圖：依據三大資產（現金、投資、儲蓄）比例動態繪製
- 資產列表（右側皆顯示箭頭）：
  - 可用現金（顯示金額）：點擊進入收支明細頁面
  - 投資理財（顯示金額）：點擊進入股票頁面
  - 銀行儲蓄（顯示金額）：點擊進入儲蓄頁面
  - 持有保險（顯示張數）：點擊進入保險頁面

## 4. 簽到頁面

### 4.1 頁面概述

- 前置路徑：於首頁點擊「簽到」後進入
- 頂部資訊列：返回按鈕、當前經驗值、可用現金、個人頭像

### 4.2 等級與薪資進度卡片

- 左側（當前狀態）：目前等級、當前稱號、當前日薪
- 中央（經驗進度）：進度條，顯示目前經驗值 / 升級所需經驗值
- 右側（下一級目標）：下一等級、下一級稱號、下一級日薪

### 4.3 簽到日曆區塊

以月曆形式呈現，顯示當月每日簽到狀態：
- 已簽到：淺綠色背景、金幣圖示、綠色打勾，不可再次點擊
- 今日未簽到：橘紅色背景、爆炸貼紙樣式、「簽到」字樣，為可互動按鈕
- 未簽到 / 未來日期：僅顯示日期數字，不可點擊

### 4.4 簽到互動流程

**步驟一：簽到確認**
- 點擊日曆上的「簽到」圖示
- 彈出確認視窗：「簽到時，將抽取命運卡。確定現在要簽到了嗎？」
- 操作：「我要簽到」（進入步驟二）/ 「取消」（關閉彈窗）

**步驟二：簽到成功與薪資發放**
- 彈出「簽到成功」視窗，顯示今日日期與獲得薪水
- 系統即時發放薪資金幣，更新日曆為「已簽到」
- 視窗下方顯示「抽取命運卡」按鈕（必須點擊，無法直接關閉）

**步驟三：命運卡判定與展示**
- 系統隨機判定：觸發風險事件、無事發生、或正面獎勵
- 顯示「命運一抽」卡片：事件插圖、描述文案、損益金額
- 操作選項：
  - 「支付金幣」（紅色按鈕）：直接扣除對應金額。若現金不足，允許餘額為負值
  - 「我有保險」（綠色按鈕）：系統檢查是否持有對應類型且效期內的保險，符合才可點擊，進入步驟四

**步驟四：保險理賠**
- 彈出「保險理賠」視窗，顯示理賠金額
- 獲得理賠金以抵銷風險損失
- 點擊「確認」關閉所有彈窗，完成今日簽到

## 5. 課程系統

### 5.1 課程首頁

- 前置路徑：於首頁點擊「進入課程」卡片後進入
- 頂部資訊列：返回按鈕、當前經驗值、可用現金、個人頭像
- 顯示方式：以「主題圓形圖示」排列各大課程主題
- 解鎖機制：
  - 已解鎖：彩色圖示與標題，可點擊進入章節列表
  - 待解鎖：灰色圖示與鎖頭，無法點擊。需完成前一主題或達到特定等級

### 5.2 章節單元列表

- 頂部資訊列：返回按鈕、當前課程主題、經驗值、可用現金、個人頭像
- **路徑式設計**：採 S 型蛇行路徑排列單元卡片，單元間以虛線連接，營造闖關體驗
- 單元卡片結構：類型標籤（小故事/小知識/小測驗/小遊戲）+ 單元標題
- 背景顏色狀態：已完成（綠色）/ 未完成（黃色）
- **懸浮導引對話框**：吉祥物與對話框指向下一個建議進行的單元
- 完成回饋：完成學習後自動跳轉回此頁，更新狀態並移動導引至下一單元

#### 章節類型

| 類型 | Icon | 說明 |
|------|------|------|
| 小故事 | 故事書 | 情境引導 |
| 小知識 | 放大鏡 | 核心觀念教學 |
| 小測驗 | 問卷板 | 選擇題測驗 |
| 小遊戲 | 遊戲搖桿 | 互動式小遊戲 |

### 5.3 小故事頁面

**步驟一：影片觀看**
- 畫面中央顯示影片播放器
- 影片必須完整播放一次，右側「下一步箭頭」才會出現

**步驟二：情境選擇**
- 顯示三個情境選項（A、B、C）與詳細敘述
- 選中選項高亮，未選中變灰色半透明
- 左側箭頭「重播影片」，右側箭頭「確認選擇」（需先選擇才可點擊）

**步驟三：結果結算**
- 顯示該選項對應的劇情發展文字
- 經驗值獎勵與星級評價（10 經驗值 = 1 顆星）
- 操作：「再看一次」返回步驟一 / 「前往下個章節」領取經驗值並返回列表

### 5.4 小知識頁面

**步驟一：影片觀看**
- 同小故事，影片必須完整播放才能進入下一步

**步驟二：完成結算**
- 顯示「完成小知識，成功獲得經驗！」與經驗值獎勵
- 操作：「再看一次」/ 「前往下個章節」

### 5.5 小遊戲頁面

**遊戲元件架構**
- 各遊戲為獨立可重用元件（位於 `features/course/games/`），僅需傳入 `onComplete(score)` callback 即可在任何頁面使用
- 遊戲元件負責遊戲邏輯與 UI，結束時回傳分數（0~100）；結算獎勵由呼叫端處理

**目前內建遊戲**
| 遊戲 | 規則 | 3星條件 | 2星條件 | 1星條件 |
|------|------|---------|---------|---------|
| 猜數字 | 猜 1~100 的數字，提示大小 | ≤3 次 | ≤5 次 | ≤8 次 |
| 記憶翻牌 | 4x4 emoji 配對 | ≤10 次 | ≤15 次 | ≤20 次 |

**步驟一：遊戲進行**
- 畫面中央為遊戲畫面
- 遊戲結束時自動進入結算

**步驟二：完成結算**
- 顯示總得分與星級獎勵對照表：
  - 1 星：金幣 +50
  - 2 星：金幣 +100
  - 3 星：金幣 +200（黃底高亮框）
- 操作：「再玩一次」/ 「前往下個章節」

### 5.6 小測驗頁面

**步驟一：題目作答**
- 顯示題號與題目內容，選項 1x4 垂直排列
- 點擊選項選中（藍框），點擊右側箭頭提交

**步驟二：即時詳解**
- 正確選項綠色邊框，錯誤選項紅色邊框
- 下方展開粉紅色詳解區塊，支援垂直滾動
- 點擊右側箭頭進入下一題或結算

**步驟三：結果結算**
- 顯示「總共答對 X 題」
- 獎勵公式：
  - 經驗值：50 +（答對題數 × 10）
  - 金幣：答對題數 × 10
- 操作：「再看一次題目」（唯讀模式）/ 「前往下個章節」

## 8. 投資理財（股票頁面）

### 8.1 頁面概述

- 前置路徑：於首頁點擊「投資理財」後進入
- 頂部資訊列：返回按鈕、當前經驗值、可用現金、個人頭像
- 導覽結構：頂部「資產儀表板」兼「分頁切換器」

### 8.2 頂部資產儀表板

- 股票總資產、已實現損益、未實現損益、總損益、可用現金
- 點擊「已實現損益」切換至對應分頁
- 點擊「未實現損益」切換至對應分頁
- 色彩規範（台股標準）：獲利/上漲紅色、虧損/下跌綠色

### 8.3 交易時段限制

- 股票買賣僅限台灣時間 16:00-21:00
- 非交易時段送出買賣請求時，後端回傳錯誤提示「目前非交易時段，交易時間為 16:00-21:00」

### 8.4 分頁一：買賣股票

**左側：股票列表**
- 顯示所有可交易股票：商品名稱/代號、成交價、漲跌金額、漲跌幅
- 點擊項目切換右側詳細面板，顯示選中標記

**右側：詳細交易面板**
- 股票資訊：大字體價格、代號、漲跌幅
- 走勢圖：當週簡易股價折線圖
- 持倉資訊：持有數量、平均成本、未實現損益
- 交易操作區：
  - 數量選擇器：輸入框 + 上下按鈕（一次增減 100）
  - 單位切換：「股」（1 股）/「張」（1,000 股）
  - 買進（綠色）：扣除現金增加持股，現金不足需提示
  - 賣出（紅色）：扣除持股增加現金，持股不足需提示

### 8.5 分頁二：未實現損益

- 儀表板「未實現損益」區塊選中，底部綠色底線
- 右側改為「買賣股票」按鈕
- 列表欄位：股票名稱/代號、庫存股數、平均成本、現在價格、參考市值、未實現損益、報酬率
- 計算公式：
  - 參考市值 = 庫存股數 × 現在價格
  - 未實現損益 =（現在價格 - 平均成本）× 庫存股數
  - 報酬率 = 未實現損益 /（平均成本 × 庫存股數）%
- 色彩：正值紅色、負值綠色

### 8.6 分頁三：已實現損益

- 儀表板「已實現損益」區塊選中，底部綠色底線
- 右側改為「買賣股票」按鈕
- 列表欄位：時間、股票名稱、交易股數、買入單價、賣出單價、損益金額、報酬率
- 色彩：正值紅色、負值綠色

### 8.8 股價更新機制

- 每日 15:00（台灣時間）由 Cloud Functions scheduled function 自動從 TWSE（台灣證券交易所）API 抓取當日真實收盤價
- 假日/休市日 API 無資料則跳過不更新
- 資料架構：
  - `config/currentStockPrices`：聚合文件，存所有股票當日價格（列表頁 1 read）
  - `stocks/{stockId}.priceHistory`：object field 存歷史收盤價（個股走勢圖 1 read）

## 12. Admin 後台管理

### 12.1 權限機制

- 使用者文件 `users/{uid}` 新增 `role` 欄位，值為 `'admin'` 時可訪問後台
- 角色指派方式：手動在 Firestore console 設定
- Firestore security rules 定義 `isAdmin()` 函式，允許 admin 對公開 collection 進行寫入
- 前端 `AdminRoute` 元件檢查使用者 role，非 admin 導回首頁

### 12.2 路由結構

- `/admin` — 後台首頁 Dashboard
- `/admin/:collection` — 各 collection 通用 CRUD 頁面（fateCards, courses, insuranceProducts, levels, stocks）
- `/admin/courses/:courseId/chapters` — 課程章節管理

### 12.3 CRUD 功能

- 通用 CRUD 頁面：表格列表 + 新增/編輯 Modal + 刪除確認
- 欄位定義由 config 設定檔驅動，各 collection 共用同一套 CRUD 邏輯
- 課程章節的 `content` 欄位使用 JSON textarea 編輯
- 文件 ID 為手動輸入（與 seed 資料一致）
- 直接透過 Firestore SDK 寫入，由 security rules 保護
