# Lesson Plan 撰寫指南

本 Skill 引導你從課綱出發，為財商學習平台撰寫完整的 lesson plan。每個 lesson plan 包含多個章節，每個章節的內容可直接用於：
1. Firestore seed 資料（`functions/src/seed/courses.ts` 格式）
2. course-video skill 的影片製作（story / knowledge 章節）

## 對象與語氣

- **目標受眾**：7～12 歲學生
- **語氣**：親切、活潑、口語化，避免艱深術語
- **角色**：為每個故事創造一個有名字的主角（如小明、小美、阿傑）

### 核心原則：像面對面上課一樣寫

你不是在寫教科書，你是一個站在學生面前的老師。所有內容都要像在「跟學生說話」，而不是「寫給學生看」。

**互動技巧（必須頻繁使用）：**
- **開場提問**：用問題勾起好奇心，再帶入知識。「你有沒有想過，為什麼存錢筒的錢放再久都不會變多？」
- **反問引導**：拋出問題讓學生思考，停頓一下再揭曉。「猜猜看，哪一個才是『需要』？……沒錯！」
- **驚奇感**：用「你知道嗎？」「很多大人都不知道喔！」製造驚喜
- **連結經驗**：從學生已知的事情出發。「你有沒有去過超市？推著購物車，看到什麼都想放進去對吧？」
- **具體化**：不說「很多錢」，說「一天一杯飲料，一個月就是三十杯！」

## 輸入

### 課綱資料

位於 `doc/curriculums/` 目錄，每個檔案代表一個階段的課綱，包含：

| 欄位 | 說明 |
|------|------|
| 堂次 | 課堂編號 |
| 課程名稱 | 該堂課的主題 |
| 學習元素 | 該堂課要教的知識點（通常 3～5 個） |

### 指定範圍

使用者會指定要撰寫哪些堂次，例如「請撰寫第 1～3 堂的 lesson plan」。

## 輸出

### 輸出位置

將 lesson plan 寫入 `doc/lesson-plans/` 目錄，檔名格式：

```
doc/lesson-plans/stage-{階段}-lesson-{堂次}.md
```

例如：`doc/lesson-plans/stage-1-lesson-01.md`

## 章節規劃原則

### 標準章節組合

每堂課建議 4～6 個章節，遵循以下順序：

```
小故事 → 小知識 → [小知識] → 小測驗 → 小遊戲
```

| 位置 | 類型 | 數量 | 目的 |
|------|------|------|------|
| 開頭 | 小故事 | 1 | 以情境引起興趣，帶出本課主題 |
| 中段 | 小知識 | 1～3 | 拆解學習元素，每個知識影片涵蓋 3～5 個知識點 |
| 中後 | 小測驗 | 1 | 檢驗學習成效，3～5 題選擇題 |
| 結尾 | 小遊戲 | 1 | 趣味收尾，鞏固記憶 |

### 學習元素分配

課綱中每堂課有 3～5 個學習元素，分配方式：

1. **選 1 個**最適合用故事情境呈現的元素 → 小故事
2. **將剩餘元素**依主題相近度分組 → 每組 1 個小知識章節
3. **從所有元素中**出 3～5 題選擇題 → 小測驗
4. 選擇適合的遊戲類型 → 小遊戲

### 內容撰寫原則

**小故事**
- 主角是學生，遇到與主題相關的日常情境
- 旁白要像在跟觀眾聊天，不是平鋪直敘描述事件。用「欸！你看！」「天哪！」「結果你猜怎麼了？」拉近距離
- 三個選項要有明確的好壞區分：1 個最佳（3 星）、1 個尚可（2 星）、1 個不好（1 星）
- 結果說明要帶出教學意義，而非單純說對錯

**小知識**
- 每個知識點配一個直覺的 emoji icon
- 標題用簡短有力的句子，可以是提問句（「什麼是記帳？」）或定義句（「需要 vs 想要」）
- **不要直接下定義！** 先用提問或情境開場，引起好奇後再解釋
- 每個知識點 3～5 句，用對話口吻寫，像老師在教室裡跟學生面對面講解
- 穿插互動語句：「你覺得呢？」「猜猜看！」「答對了！」「是不是很驚人？」
- 善用具體數字和生活化舉例，不說抽象概念
- **配圖應頻繁更換**：講到新主題、新物品、新例子就配新圖。一支影片十幾到數十張配圖是正常的。
- **旁白與畫面必須協調**：如果旁白正在提問讓學生猜答案，畫面上絕對不能顯示答案。答案應在下一個場景揭曉。

**國際化原則（所有章節適用）**
- 不使用特定國家的教育體制用語（如「國小」「國中」），改用年齡範圍
- 不使用特定國家的貨幣或價格（如「50 元」「新台幣」），金額用平台的「金幣」或通用數量描述
- 使用各國通用的生活情境（超市、書店、文具店），避免特定文化場景（如夜市、便利商店集點）
- 食物、節日、地名等應選擇全球通用的例子，或可輕易替換的泛稱

**小測驗**
- 題目涵蓋該堂課所有學習元素
- 選項要有迷惑性但不刁鑽
- 詳解要補充知識，不只說「因為 B 是對的」
- 正解分布要均勻（不要全部都是 B）

**小遊戲**
- 交替使用 `guessNumber` 和 `memoryFlip`
- 奇數堂用 guessNumber、偶數堂用 memoryFlip（或反之）

### Before / After 範例

以下對比說明「扁平教科書式」與「面對面教學式」的差異。撰寫時必須對齊 After 的風格。

**小知識 — 知識點範例**

> ❌ Before（教科書式，不要這樣寫）：
>
> **需要 = 生活中不可缺少的東西** 🍚
> 像是每天的三餐、喝的水、上學穿的制服，這些沒有的話生活會出問題。
>
> **想要 = 有的話很開心，沒有也沒關係** 🧸
> 像是新玩具、遊戲點數、偶像的專輯，有了會很快樂，但沒有也不會怎樣。

> ✅ After（面對面教學式，要這樣寫）：
>
> **需要 vs 想要，到底差在哪？** 🍚
> 你有沒有去過超市？推著購物車，看到什麼都想放進去對吧？但你有沒有發現，爸爸媽媽最後結帳的時候，有些東西又被放回去了？那是因為花錢其實分兩種：「需要」和「想要」。需要就是沒有的話會出問題的東西，像是每天的午餐、喝的水、上學穿的制服。想要呢？就是有了很開心，但沒有其實也不會怎樣的東西，像是最新的遊戲卡、可愛的貼紙。你猜猜看，「新書包」和「第 10 盒彩色筆」，哪個是需要、哪個是想要？
>
> **記帳就是幫錢拍照！** 📒
> 你有沒有過這種經驗：明明才剛拿到零用錢，怎麼一下子就花完了？錢好像長了腳自己跑掉！其實不是錢跑掉了，是你忘記它們去了哪裡。這時候「記帳」就派上用場啦！記帳就像幫每一塊錢拍照、做紀錄。你買了一杯飲料？寫下來。買了一支鉛筆？寫下來。一個禮拜後回頭看，哇！原來我花最多錢的地方是這裡！是不是很像偵探在找線索？

**小故事 — 旁白範例**

> ❌ Before（平鋪直敘）：
>
> 小琪走進文具店，發現書包的拉鍊壞了需要換新，但也看到一組超可愛的彩色筆。

> ✅ After（對話式）：
>
> 小琪一走進文具店，眼睛馬上亮了起來——哇！那邊有一組超～可愛的彩色筆！但等等，她低頭看看自己的書包……拉鍊壞掉了，課本都快掉出來了啦！怎麼辦？兩個都想要，但口袋裡的錢只夠買一個！

## 操作步驟

1. **讀取課綱**：開啟 `doc/curriculums/stage-{N}.md`，找到指定堂次
2. **分析學習元素**：列出該堂課的 3～5 個學習元素，決定分配方式
3. **規劃章節**：確定章節數量、類型、標題
4. **撰寫內容**：依上方格式逐章撰寫
5. **自我檢查**：
   - [ ] 所有學習元素都有被涵蓋？
   - [ ] 故事情境對 7～12 歲學生來說夠生活化？
   - [ ] 內容是否國際化（無特定國家的貨幣、教育體制、文化場景）？
   - [ ] 知識點有用提問或情境開場，而不是直接下定義？
   - [ ] 每個知識點至少 3 句，讀起來像老師在講話、不像教科書？
   - [ ] 內容中有穿插互動語句（猜猜看、你覺得呢、是不是很驚人）？
   - [ ] 故事旁白像在跟觀眾聊天，而不是平鋪直敘？
   - [ ] 測驗正解分布均勻？
   - [ ] 影片腳本欄位完整（場景、文字、Icon）？
6. **輸出**：寫入 `doc/lesson-plans/stage-{N}-lesson-{堂次}.md`
